Especificações — Autenticação e Hub

Hub de apps por área · SSO · Discovery ERP (Datasul, TMS, Mercanet)

Cada spec inclui tasks e botão Exportar MD (Spec → Histórias → Tasks) para implementação.

Arquitetura atual: Login portal = LDAP/JWT (/acesso). MSAL Azure = apenas Calendar e Teams (Graph). SignalR hubs usam JWT portal.

1. Hub com Aplicações por Área de Negócio Protótipo

História: US-AUTH-001

Objetivo

Centralizar launch de apps desenvolvidas pela TI (e futuramente ERP) por área: Comercial, Operações, Finanças, TI, RH.

Estado atual

  • /servicos/acesso-sistemas — legacy mock, botões href="#"
  • Hubs internos existem: /servicos/rh, /servicos/ti (rotas portal)
  • Padrão: *-hub.ts + *HubPage.tsx

Arquitetura proposta

GET /apps/catalog?area=comercial|operacoes|financeiro|ti
→ { id, name, description, icon, launchUrl, ssoType, requiredRoles }

ssoType: none | deeplink | msal | saml | oidc

Frontend F4

  • Substituir legacy por React hub (modelo TiHubPage)
  • Launch: window.open ou SSO popup
  • Admin: seção em Backend Config para cadastro apps

RBAC

Filtrar apps por MeDto.roles e departamento; audit log de acessos.

2. Framework SSO Reutilizável Parcial

História: US-AUTH-002

Objetivo

Generalizar padrão link-account (Calendar/Chat) para outros apps Azure ou SAML/OIDC.

Estado atual

  • azureMsal.ts — popup + silent token
  • POST /calendar/link-account, POST /chat/link-account
  • Config: azure_ad.client_id, azure_ad.tenant_id

Extensão F4

  • Token broker backend para apps não-Graph
  • SAML/OIDC proxy se ERP não federado Entra ID
  • Componente AppLaunchGate (similar LoopAccessGate)

3. Datasul — Discovery Bloqueado

História: US-AUTH-003 (Spike)

Dependência: avaliar com fornecedor — não implementável até discovery.

Perguntas fornecedor

  • API REST/SOAP disponível? Documentação?
  • SSO via SAML/OIDC ou credencial própria?
  • Federado com Entra ID (Azure AD)?
  • Ambientes homolog/prod e VPN necessária?
  • Quais módulos prioritários (financeiro, estoque, compras)?

Entregável spike

Documento ADR com opções, esforço estimado, recomendação — output R4.

2. Solicitação de Férias Integrado

Histórias: US-RM-002, US-RM-003

Objetivo

Colaborador solicita férias, consulta saldo e acompanha status até aprovação RM.

Fluxo

  1. Colaborador abre módulo férias → consulta saldo (/rh/leave/balance)
  2. Preenche período → POST /rh/leave/requests
  3. Portal exibe status (pendente/aprovado/rejeitado)
  4. Backend write-back RM (validar homolog)

Regras

  • Saldo insuficiente bloqueia envio
  • Gestor/RH aprovam conforme política interna

Fora de escopo

Parametrização de política de férias no RM (admin folha).

3. Ajuste de Ponto Novo

Histórias: US-RM-004, US-RM-007, US-RM-008

Objetivo

Colaborador solicita correção no espelho de ponto; gestor e RH aprovam; RM recebe movimentação.

Atores

Colaborador, Gestor, RH Admin

Fluxo principal

Colaborador → seleciona dia no espelho → informa motivo + horário correto
    → Gestor aprova/rejeita
    → RH valida
    → Worker/API envia ajuste ao RM
    → Espelho atualizado no próximo sync

API proposta

POST   /rh/ponto/adjustments
GET    /rh/ponto/adjustments?status=
PATCH  /rh/ponto/adjustments/{id}/approve  (gestor)
PATCH  /rh/ponto/adjustments/{id}/validate (RH)

Integração RM

Tabelas batida/justificativa (ABATFUN, AAFHTFUN — validar com equipe RM).

Estado atual

Ponto é somente leitura — PontoEletronicoPage.tsx sem write.

5. Requisição — Promoção Novo

História: US-RM-010

Objetivo

Gestor/RH abre requisição de promoção com novo cargo, salário, centro de custo.

Atores

Gestor (solicita), RH (valida), Diretoria (aprova — se aplicável)

Campos formulário

  • Colaborador, cargo atual, cargo proposto, salário proposto, CC, motivo, anexos
POST /rh/requests/movimentacao  { tipo: "promocao", ... }
GET  /rh/requests/movimentacao?tipo=promocao&status=

Estado atual

Legacy /servicos/solicitacoes-rh — card mock offline, sem API.

6. Requisição — Desligamento Novo

História: US-RM-011

Campos

8. Ciclo de Vida da Vaga até Contratação Novo

Histórias: US-RM-013, US-RM-014

Objetivo

Abrir requisição de vaga, acompanhar pipeline (triagem → entrevista → proposta → contratação).

Estados pipeline (MVP)

aberta → em_triagem → entrevista → proposta → contratada | cancelada

API proposta

POST   /rh/vagas
GET    /rh/vagas?status=
PATCH  /rh/vagas/{id}/status
POST   /rh/vagas/{id}/candidatos
PATCH  /rh/vagas/{id}/contratar

UI

Kanban ou lista com filtros; notificações gestor/RH em mudança de status.

Fora de escopo MVP

Integração job boards externos; admissão digital completa.

1. Painel Administração de Comunicados Parcial

História: US-INTRA-001

Objetivo

Admin/Comunicação cria, edita, publica e arquiva comunicados oficiais, departamentais e urgentes.

Estado atual

  • Listas integrated: /comunicados/oficiais, departamentais, urgentes, arquivo
  • Editor ComunicadoEditorPage.tsx — protótipo; POST /comunicados existe
  • Filtros e busca na listagem — UI-only (sem handler)

Entregáveis F1

  • CRUD completo com preview
  • Métricas de leitura (visualizações, % lidos)
  • Workflow rascunho → publicado → arquivado
POST   /comunicados
PATCH  /comunicados/{id}
GET    /comunicados/{id}/metrics

2. Comunicação Parcial

História: US-INTRA-002

Objetivo

Canal unificado de comunicação interna — feed, comunicados, hubs.

Estado atual

  • Feed híbrido: API posts + cards HTML estáticos (enquetes, notícias demo)
  • Hub comunicados com fallback estático
  • Grupos parcialmente integrados

Meta F1–F3

Reduzir protótipos visíveis; comunicados como fonte oficial; feed para social/engajamento.

3. Aniversariantes Parcial

História: US-INTRA-003

Objetivo

Listar aniversariantes próximos 60 dias; permitir parabenizar.

Estado atual

  • GET /people/birthdays?days=60
  • Legacy page + hook useBirthdays no calendário
  • Botão "Parabenizar" — href="#" (não wired)

Entregáveis F1

  • Parabenizar → post no feed ou notificação
  • Sync RM + Graph com fallback
  • Migrar legacy → React (opcional F1)

4. Feed de Notícias Parcial

História: US-INTRA-004

Objetivo

Notícias internas unificadas — sem depender de cards estáticos no feed.

Estado atual

  • Hub /noticias — links hardcoded para âncoras do feed
  • Sem CMS/API dedicada de notícias

Opções F3

  1. Notícias como tipo de post no feed API (kind: news)
  2. Comunicados departamentais como fonte de notícias
  3. CMS leve (admin cria notícia → aparece feed + hub)

5. Como me sinto Parcial

História: US-INTRA-005

Objetivo

Colaborador registra humor diário; RH visualiza tendências agregadas.

Estado atual

  • MoodCheckCard no feed — GET/POST /mood
  • Texto feedback randomizado local (mood-feedback.ts)
  • Desabilitado em modo mock

Entregáveis F1

GET /mood/analytics?period=week|month  (RH Admin)
Dashboard: distribuição por nível, tendência, anonimizado

6. Envio de Feedback Novo

História: US-INTRA-006

Objetivo

Canal formal de feedback (sugestão, elogio, reclamação) — distinto do mood check.

Atores

Colaborador, RH, Gestor (triagem)

Fluxo

Colaborador → formulário (categoria, texto, anônimo?)
    → Fila triagem RH/Gestor
    → Status: recebido → em_análise → respondido → encerrado
POST /feedback
GET  /feedback?status=  (RH)
PATCH /feedback/{id}/respond

UI proposta

Rota /feedback ou item em Serviços RH; link no hub intranet.

7. Indicadores por Área (Finanças, Operações, Comercial) Protótipo

História: US-INTRA-007

Objetivo

Linkar painéis/indicadores já criados por área — acessos dedicados no portal.

Estado atual

  • /analytics — KPIs portal (mock parcial)
  • /loop, /pulse — dashboards mock com times Comercial/Operações
  • Sem rotas /areas/financeiro etc.

Entregáveis F3

  • Hub por área: Finanças, Operações, Comercial
  • Tiles linkando apps/dashboards existentes (URLs externas ou internas)
  • RBAC — só área autorizada vê seu hub

Relacionado a Hub apps (EPIC-AUTH).
